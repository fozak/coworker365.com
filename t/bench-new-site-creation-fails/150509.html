This issue got fixed. The following are the steps that I did to make it work.
1.) Go to frappe-bench(or the folder that you have init using bench command). Then uninstall using ‚Äú~/frappe-bench/env/bin/python -m pip uninstall mysqlclient‚Äù. (The reason why this error occurred is the mysqlclient that came as part of the installation didn‚Äôt match the arch type of m1.)
2) Then try installing ‚Äú~/frappe-bench/env/bin/python -m pip install mysqlclient --no-cache-dir‚Äù. To download a PIP package rather than using cache stored locally.
However when trying to install this mysqlclient i got an issue that openssl is not available in the machine. i had to do ‚Äúbrew install openssl‚Äù followed by step 2.
After that ‚Äúbench new-site site.name‚Äù works without issue.
