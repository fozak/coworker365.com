# Steps to Install ERPNext via Docker

# 1. Update System and Install Docker + Docker Compose

Run the following to install dependencies:

```bash

sudo apt update && sudo apt upgrade -y

sudo apt install curl git -y

curl -fsSL https://get.docker.com | bash

sudo apt install docker-compose -y

# Check versions

docker --version

docker compose version

``# 2. Open Essential Ports

Allow HTTP, HTTPS, and SSH traffic:

```bash

sudo ufw allow 80/tcp

sudo ufw allow 443/tcp

sudo ufw allow 22/tcp

sudo ufw enable

sudo ufw status

```

# 3. Clone the ERPNext Docker Repository

```bash

git clone GitHub - frappe/frappe_docker: Docker images for production and development setups of the Frappe framework and ERPNext

cd frappe_docker

```

# 4. Configure the Production Docker Setup

- Make a new working directory for your ERPNext project:

```bash

mkdir ~/erpnext-docker

cd ~/erpnext-docker

```

- Copy example production compose files or create your own `docker-compose.yml`. The official `frappe/frappe_docker` repo provides sample configurations for production use. Adjust values for DB passwords and domain as needed.

# 5. Launch Docker Containers

```bash

docker compose -p erpnext -f docker-compose.yml up -d

```

This command will initialize all services (ERPNext, MariaDB, Redis, etc.) in detached mode

# 6. Create a New ERPNext Site

After containers are up, create your ERPNext site:

```bash

docker compose exec backend bench new-site --mariadb-root-password <db_root_password> --admin-password <erp_admin_password> <site_name>

docker compose exec backend bench --site <site_name> install-app erpnext

```

Note - Replace `<db_root_password>`, `<erp_admin_password>`, and `<site_name>` accordingly (e.g., `erp.yourdomain.com`)

# 7. Access ERPNext

- Open your browser to `http://<your_vps_ip>:8080` (or your configured domain if set up with DNS/Reverse Proxy).

- Default login: user `Administrator`, pass `admin` (change on first login!)
