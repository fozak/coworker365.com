umair June 6, 2017, 12:23pm 2

If you have stock of raw-material item’s available, and if BOM for the sub-assembly item’s valuation is set to be calculated on “Valuation Rate”, then clicking on “Update Cost” will update raw-material and BOM prices. Following this approach, go on updating cost from raw-material to sub-assembly to FG item’s BOM.

I have the exact same issue here, it’s very frustrating and seems like it should be a simple fix (and an important part of an ERP system).

I tried to create a new field in the Item form “BOM cost” which fetched the value from the BOM form, Total cost, but this didn’t work either. Anyone have any ideas why even that doesn’t work?

I also found that if you don’t have any stock for items in a BOM, it will update the BOM cost correctly after changing the item valuation rate. But if you have stock, you need to go and also update the Stock Entry that contains the items, because the BOM cost will calculate based on the stock entry number, not the latest item number. Hope that makes sense to everyone.
