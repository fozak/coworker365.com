Bug Report: “Save” button becomes invisible when Assignment Rule conditions are long

Business Apps Frappe Helpdesk
bug
hailm December 4, 2025, 9:40am 1

Hi team,

I’ve encountered a usability issue when creating or editing an Assignment Rule. When the rule contains many conditions, the form expands vertically and the Save button is pushed completely out of view. There is no scroll continuation or fixed footer, so users cannot reach the Save button at all.

image2040×1528 334 KB

As a result, the rule cannot be updated or saved, which makes the entire feature unusable in cases where long condition sets are required (common for warehouse routing rules).

This is a critical issue because Assignment Rules are an essential part of our automated workflow. Without the ability to save the rule, agents cannot configure or adjust routing logic, leading to operational delays and manual intervention.

Could we adjust the layout so that the Save button always remains accessible (e.g., sticky footer, auto-scroll, or collapsible condition groups)?

Thank you!
