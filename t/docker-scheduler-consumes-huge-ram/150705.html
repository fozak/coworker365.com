Running jobs entails pushing job data into some sort of pipeline and assuring results get where they need to get. Depending on what and how this is done, this could imply lots of data and also caching, indexing, etc. Running scheduler jobs generally takes place for long-running tasks which cannot be done while serving a request, so they get pushed to be run at other times. It‚Äôs easy to make assumptions about things, although things in reality are different.
